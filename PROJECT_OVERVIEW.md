## 1. 프로젝트 소개

### 투자위원회(IC)란?

스타트업이나 기업에 큰 돈을 투자하기 전, 실제 금융회사에서는 반드시 **투자위원회(Investment Committee, IC)** 라는 회의를 엽니다.

이 회의에서는 다양한 전문가들이 모여 딜의 장단점을 토론합니다:

| 역할 | 하는 일 |
|------|---------|
| 재무 분석관 | 숫자를 보고 수익성 판단 |
| 리스크 심사역 | 위험 요소 발굴 |
| 강세론 위원 | 왜 투자해야 하는지 주장 |
| 약세론 위원 | 왜 투자하면 안 되는지 반박 |
| 투자위원장 | 전체 토론을 듣고 최종 결정 |

이 과정은 보통 **수 주~수 개월**이 걸립니다. 자료 수집, 분석, 보고서 작성까지 포함하면 엄청난 시간과 인력이 필요합니다.

### 이 시스템이 하는 일

> **이 시스템은 위의 투자위원회 과정 전체를 AI로 시뮬레이션합니다.**

사용자가 기업명과 투자 금액을 입력하면:
1. AI가 공시 데이터를 자동 수집
2. 여러 AI가 각자 역할을 맡아 토론
3. 금융 충격 시뮬레이션 (금리 인상, 환율 하락 등)
4. 최종 투자 검토 보고서(IC Memo) 자동 생성

수 주 걸리던 작업이 **수 분** 안에 초안으로 완성됩니다.

## 2. 왜 만들었는가?

### 기존 방식의 문제

실제 투자 분석은 이런 과정을 거칩니다:

```
① 애널리스트가 DART(금융감독원 공시시스템)에서 수십 개 문서 수집
② 엑셀로 재무 모델 구축 (수 일)
③ 회의 준비 자료 작성 (수 일)
④ 투자위원회 회의 (수 시간)
⑤ 회의 결과를 보고서로 정리 (수 일)
```

**문제점:**
- 반복적이고 시간이 많이 걸리는 작업이 많음
- 담당자의 주관적 관점에 치우칠 수 있음
- 다양한 시나리오(금리 인상, 경기 침체 등) 검토가 번거로움
- 보고서 형식이 표준화되지 않아 일관성 부족

### AI로 해결하는 것

| 기존 문제 | AI 해결책 |
|-----------|-----------|
| 자료 수집에 며칠 소요 | DART API 자동 수집 (수 초) |
| 한 명의 관점에 치우침 | 여러 AI가 각자 다른 역할로 토론 |
| 충격 시나리오 계산이 복잡 | 수학적 시뮬레이션 자동 계산 |
| 보고서 작성에 수 일 소요 | 자동 생성 (수 분) |

> **중요:** 이 시스템은 사람을 대체하는 것이 아니라, **초안 작성과 다각도 검토를 도와주는 도구**입니다. 최종 판단은 항상 사람이 합니다.

## 4. Multi-Agent란 무엇인가?

### 쉬운 비유

> 한 명의 천재 AI에게 모든 것을 물어보는 대신,
> **각자 전문 분야가 다른 여러 AI가 회의를 여는 구조**입니다.

마치 실제 회사에서 재무팀, 법무팀, 전략팀이 각자 의견을 내고 토론하는 것과 같습니다.

### 왜 이렇게 하는가?

한 명의 AI에게 "이 기업에 투자해야 해?"라고 물으면, 그 AI는 **하나의 관점**만 제시합니다. 하지만 실제 투자 결정은 다양한 각도에서 검토해야 합니다.

```
❌ 나쁜 방식:
AI에게 "삼성전자 투자해야 해?"
→ AI: "좋은 것 같습니다. 투자하세요."
(한 관점, 근거 불명확)

✅ 이 시스템의 방식:
여러 AI가 각자 다른 역할로 분석 후 토론
→ 더 균형 있고, 다각도의 결론
```

### 각 AI의 역할

#### 🔍 데이터 수집 AI (Data Collection Agent)
> **역할:** 회의를 시작하기 전, 필요한 자료를 모두 정리하는 담당

- DART에서 가져온 삼성전자 재무제표 정리
- 매출, 영업이익, 주요 사업부 현황 파악
- 다른 AI들이 분석하기 좋게 팩트를 구조화

#### 📊 재무 분석 AI (Financial Analysis Agent)
> **역할:** 숫자 전문가 — 수익이 나는지 계산

- "삼성전자의 영업이익률이 2.5%인데, 이는 역대 최저 수준입니다"
- "2024년 반도체 업황 회복 시 영업이익 30조원 이상 회복 가능성이 있습니다"
- 투자 수익률(IRR, MOIC) 관점에서 딜 타당성 평가

#### 🛡️ 리스크 심사 AI (Risk Agent)
> **역할:** 비관론자 — 모든 위험 요소를 찾아내는 담당

- "중국 반도체 수출 규제가 강화될 경우 매출 타격이 예상됩니다"
- "메모리 다운사이클이 예상보다 길어질 가능성이 있습니다"
- 충격 시나리오 영향 분석

#### 🟢 강세론 AI (Bull Agent)
> **역할:** 낙관론자 — 왜 투자해야 하는지 최선의 논거를 구성

- "HBM(고대역폭 메모리) AI 수요 급증으로 수혜 가능성이 있습니다"
- "반도체 사이클 바닥 통과 후 주가 상승 여력이 있습니다"
- 투자의 긍정적 측면 강조

#### 🔴 약세론 AI (Bear Agent)
> **역할:** 비판론자 — 왜 투자하면 안 되는지 반론 구성

- "현재 주가에 이미 회복 기대가 반영돼 상승 여력이 제한적일 수 있습니다"
- "소수 지분 투자의 경우 거버넌스 영향력이 없습니다"
- 강세론의 모든 가정에 의문을 제기

#### 👔 투자위원장 AI (Chairman Agent)
> **역할:** 사회자 겸 결정자 — 모든 토론을 듣고 최종 결의문 작성

- 각 AI의 투표 집계 (찬성/반대/조건부)
- 양쪽 논거를 균형 있게 정리
- 조건부 승인 시 구체적인 조건 제시
- 밸류에이션 충분성, 거버넌스 권리, 전략적 적합성 종합 판단

### 토론 구조

```
[데이터 수집 AI]
       │ 팩트 정리 완료
       ▼
┌──────────────────────────────────────┐
│          병렬 심의 (동시 진행)        │
│  [재무 분석 AI] → 투표: CONDITIONAL  │
│  [리스크 AI]   → 투표: REJECT        │
│  [강세론 AI]   → 투표: APPROVE       │
│  [약세론 AI]   → 투표: CONDITIONAL   │
└──────────────────┬───────────────────┘
                   │
                   ▼
           [투자위원장 AI]
           투표 결과 집계:
           APPROVE 1 / CONDITIONAL 2 / REJECT 1
           → 최종 결정: CONDITIONAL (조건부 승인)
           → 조건: "분기별 실적 모니터링 필수"
```

## 5. RAG — AI가 문서를 찾아보는 방법

### RAG가 뭔가요?

> **비유:** RAG는 AI에게 "오픈북 시험"을 치르게 하는 기술입니다.

일반 AI는 학습할 때 배운 것만 대답합니다 (암기 시험).
RAG를 쓰면 AI가 **관련 문서를 먼저 찾아보고** 그것을 바탕으로 대답합니다 (오픈북 시험).

```
❌ RAG 없이:
사용자: "삼성전자 2023년 영업이익은?"
AI: "제가 학습한 데이터에 따르면 약 6조원입니다."
(틀릴 수도 있음, 구체적 근거 없음)

✅ RAG 사용 시:
① DART 사업보고서에서 관련 내용 검색
② 발견: "영업이익 6,566,976,000,000원 (2023)"
③ AI: "DART 2023 사업보고서 기준, 영업이익은 약 6.57조원입니다."
(출처 명확, 신뢰도 높음)
```

### 왜 Hybrid RAG를 쓰는가?

이 시스템은 두 가지 검색 방법을 **동시에** 사용합니다:

#### 방법 1: 의미 기반 검색 (Dense Retrieval)
> "의미가 비슷한 문장을 찾는 검색"

예: "수익성이 좋다"로 검색하면 "영업이익률이 높다"는 문장도 찾아냄
- 장점: 의미적으로 유사한 내용 잘 찾음
- 단점: **정확한 숫자나 기업명** 매칭에 약함

#### 방법 2: 키워드 기반 검색 (BM25)
> "정확히 일치하는 단어를 찾는 검색"

예: "HBM3E"로 검색하면 정확히 "HBM3E"가 포함된 문장만 찾음
- 장점: 금융 용어, 기업명, 수치 정확 매칭
- 단점: 의미적 유사성 파악 못함

#### Hybrid = 두 방법을 합침

```
검색어: "삼성전자 영업이익"
         │
    ┌────┴────┐
    │         │
의미 검색    키워드 검색
"수익성"     "영업이익"
"이익률"     "삼성전자"
    │         │
    └────┬────┘
         │
    결과 통합 & 순위 재조정
    (Reciprocal Rank Fusion)
         │
    가장 관련성 높은 문단 5개 추출
         │
    AI 에이전트에게 전달
```

### 기업 정보 오염 방지

중요한 기능: **삼성전자를 분석할 때 다른 회사 정보가 섞이지 않도록** 필터링합니다.

```python
# 검색 시 자동으로 기업명 필터 적용
metadata_filter = {"company_name": "삼성전자"}
# → 삼성전자 관련 문서만 검색 결과에 포함
```

## 6. 충격 시뮬레이션 — 가상 위기 실험

### 이게 뭔가요?

> **비유:** 비행기 조종사가 시뮬레이터에서 위기 상황을 연습하듯,
> 투자도 "만약 이런 상황이 오면 수익이 어떻게 변하나?" 미리 실험합니다.

### 세 가지 충격 시나리오

#### 📈 금리 인상 (Interest Rate Shock)
> "중앙은행이 금리를 올리면?"

금리가 오르면:
- 기업들이 대출 이자를 더 내야 함 → 수익성 악화
- 투자자들이 채권으로 자금 이동 → 주식 가치 하락
- 미래 수익의 현재 가치가 줄어듦 (할인율 상승)

```
예시: 금리 +200bp (2% 포인트) 인상 시
  기준 IRR: 16.0%  → 충격 후 IRR: 11.2%
  기준 MOIC: 2.10x → 충격 후 MOIC: 1.78x
```

#### 💱 환율 하락 (FX Shock)
> "원화 대비 달러가 강해지면?"

해외 매출 비중이 높은 기업(삼성전자 약 55% 해외 매출)은 환율 변동에 민감합니다.

```
예시: 원화 10% 약세 시
  해외 매출의 원화 환산액 감소
  → 전체 매출에 영향
```

#### 📉 시장 침체 (Market Downturn)
> "경기가 전반적으로 나빠지면?"

주식 시장 전체가 하락하는 시나리오입니다.

### 토네이도 차트

어떤 요인이 수익률에 가장 큰 영향을 미치는지 시각화합니다:

```
IRR에 대한 민감도 분석
────────────────────────────────────────
EV/Revenue 배수  ████████████████ (가장 큰 영향)
매출 성장률      ████████████
금리 변화        ████████
환율 변화        █████
────────────────────────────────────────
"EV/Revenue 배수가 가장 중요한 변수"
```

## 7. 실제 사용 예시 — 삼성전자

### Step 1: 입력

웹 화면 왼쪽 사이드바에서:

```
기업명: 삼성전자
산업: 반도체 / 전자
투자 단계: 상장사 (Mega-cap)
투자 금액: $100M (약 1,300억 원)
충격 시나리오: 금리 200bp 상승, HBM 수요 급감
```

### Step 2: 자동 데이터 수집

```
✓ DART 공시 데이터 확인 중: 삼성전자 …
✓ DART 데이터 인덱싱 완료: 38개 청크
✓ DART 재무 추출: 매출 258.9조원, 영업이익률 2.5%
✓ 기업 분류: 초대형 상장사 (Mega-cap) | 목표 IRR 8–25%, MOIC 1.2–4.0x
```

### Step 3: AI 토론 진행

각 AI가 DART 실제 데이터를 바탕으로 분석:

**데이터 수집 AI:**
> "삼성전자(주) 2023년 매출액 258.9조원, 영업이익 6.6조원. 반도체 부문 HBM 생산 확대 추진 중. 결산월: 12월. 대표이사: 전영현, 노태문."

**재무 분석 AI:**
> "2023년 영업이익률 2.5%는 역대 최저 수준일 가능성이 있습니다. 다만 메모리 사이클 특성상 회복 시 레버리지 효과가 클 수 있습니다. 현재 EV/EBITDA는 역사적 저점 근처로 해석될 여지가 있어 밸류에이션 매력이 있을 수 있습니다."

**리스크 AI:**
> "소수 지분 투자($100M vs. 시가총액 약 400조원)로 거버넌스 영향력이 사실상 없습니다. 중국 반도체 수출 규제 강화 시 HBM 매출에 부정적 영향이 우려됩니다."

**강세론 AI:**
> "AI 반도체 수요 급증으로 HBM 공급 부족이 지속될 가능성이 있습니다. 반도체 사이클 반등 시 영업이익 30조원 이상 회복 기대가 있습니다."

**약세론 AI:**
> "주가에 이미 회복 기대가 일부 반영됐을 가능성이 있어 추가 상승 여력이 제한적일 수 있습니다. $100M 규모는 삼성전자 시총 대비 0.025%로 투자 중요성이 미미할 수 있습니다."

**최종 투표:**
```
재무 분석: CONDITIONAL
리스크:    CONDITIONAL
강세론:    APPROVE
약세론:    CONDITIONAL
→ 최종: CONDITIONAL (조건부 승인)
```

### Step 4: 시뮬레이션 결과

```

금리 +100bp:     IRR 13.5%  MOIC 1.89x
금리 +200bp:     IRR 11.2%  MOIC 1.68x  ← 사용자가 입력한 시나리오
금리 +300bp:     IRR  9.1%  MOIC 1.49x
시장 -20%:       IRR 10.8%  MOIC 1.64x
시장 -40%:       IRR  5.3%  MOIC 1.21x
```

## 9. 사용한 기술 설명

### 🌐 Streamlit — 웹 화면을 만드는 도구
> "Python 코드 몇 줄로 웹페이지를 만들 수 있는 도구"

일반적으로 웹사이트를 만들려면 HTML, CSS, JavaScript 등 별도 언어를 배워야 합니다. Streamlit은 Python만으로 인터랙티브한 웹 앱을 만들 수 있게 해줍니다. 데이터 과학자와 AI 연구자들이 많이 사용합니다.

### 🤖 Claude API (Anthropic) — AI 엔진
> "이 시스템의 두뇌. 각 AI 위원의 실제 분석을 담당"

Anthropic이 만든 Claude AI를 API(프로그래밍 방식)로 연결해 사용합니다. 각 AI 위원이 분석 결과와 의견을 생성할 때 Claude가 실제로 작동합니다. 사람이 글을 쓰는 것처럼 자연스러운 한국어 분석 보고서를 생성합니다.

### 🔗 LangGraph — AI 토론 순서를 관리하는 도구
> "여러 AI가 순서대로 대화하도록 교통 정리하는 도구"

6명의 AI 위원이 무작위로 말하면 혼란스럽겠죠? LangGraph는 "데이터 수집 AI 먼저 → 재무/리스크/강세/약세 AI 동시에 → 마지막에 위원장 AI" 순서를 보장합니다. 일종의 회의 진행 소프트웨어입니다.

### 🗄️ ChromaDB — 문서 검색 데이터베이스
> "문서들을 의미 기반으로 저장하고 검색하는 특수 데이터베이스"

일반 데이터베이스는 정확한 키워드로만 검색됩니다. ChromaDB는 텍스트의 의미를 숫자로 변환(임베딩)해 저장하므로, "수익성"으로 검색해도 "영업이익률"이 포함된 문서를 찾을 수 있습니다.

### 📊 Plotly — 인터랙티브 차트 도구
> "마우스로 조작할 수 있는 동적 차트를 만드는 도구"

정적인 그림 대신, 마우스를 올리면 수치가 표시되고, 확대/축소가 가능한 차트를 만듭니다. 토네이도 차트, 시나리오 비교 차트 등 모든 시각화에 사용됩니다.

### 📡 DART API — 한국 금융 공시 데이터
> "금융감독원의 공식 기업 공시 데이터를 자동으로 가져오는 연결통로"

DART(Data Analysis, Retrieval and Transfer System)는 금융감독원이 운영하는 전자공시 시스템입니다. 상장기업은 분기·반기·연간 재무제표, 사업보고서 등을 의무적으로 공시합니다. 이 API를 통해 삼성전자의 실제 재무 데이터를 자동으로 가져옵니다.

### 🧮 BM25 — 키워드 검색 알고리즘
> "구글 검색처럼 단어가 정확히 일치하는 문서를 찾는 알고리즘"

"HBM3E"나 "00126380"(삼성전자 DART 고유번호) 같은 정확한 금융 용어나 코드번호를 찾을 때 필수적입니다. 의미 기반 검색만으로는 정확한 숫자나 고유 코드를 찾기 어렵습니다.

## 11. 현재 한계점

실제 AI 프로젝트를 솔직하게 평가합니다. 이 시스템이 **완벽하지 않은** 이유들:

### AI 관련 한계

**① AI 환각(Hallucination) 가능성**
> AI가 없는 사실을 만들어낼 수 있습니다.

DART 데이터가 충분하지 않은 기업의 경우, AI가 학습 데이터 기반으로 추정치를 사실처럼 서술할 수 있습니다. 모든 AI 출력은 반드시 사람이 검토해야 합니다.

**② 강한 단정 표현 잔존 가능성**
> 헤지 언어를 지시했지만, AI가 여전히 단정형 표현을 쓸 수 있습니다.

"~일 가능성이 있습니다"를 쓰라고 지시해도 항상 따르지는 않습니다. 프롬프트 엔지니어링의 본질적 한계입니다.

### 재무 모델 관련 한계

**③ 시뮬레이션은 단순화된 모델**
> 현실의 복잡한 재무 관계를 일부만 반영합니다.

실제 재무 모델은 수백 개의 가정과 변수를 포함합니다. 이 시스템의 충격 시뮬레이션은 3가지 주요 변수(금리, 환율, 시장)만 고려합니다.

**④ 소수 지분 투자의 특수성 미반영**
> 삼성전자에 $100M 투자는 0.025% 지분으로, 경영 참여가 불가능합니다.

현재 시뮬레이션은 이 투자 규모의 특수성을 완전히 반영하지 못합니다. IRR 계산은 수익률 비율만 계산하며, 실제 주식 블록딜 가격 영향, 유동성 할인 등은 반영되지 않습니다.

**⑤ 미래 예측의 한계**
> 모든 수익률 계산은 현재 가정에 기반하며, 실제 미래와 다를 수 있습니다.

IRR 16%, MOIC 2.1x는 "지금 설정한 가정이 맞다면"이라는 조건 하의 수치입니다.

### 데이터 관련 한계

**⑥ DART 데이터 의존성**
> DART에 공시되지 않은 기업(비상장사, 해외기업)은 자동 수집 불가.

**⑦ 실시간 시장 데이터 없음**
> 현재 주가, 실시간 뉴스, 애널리스트 리포트 등은 수집되지 않습니다.

### 단기 개선

- 🔲 **소수 지분 투자 특화 모델** — 블록딜 유동성, 지분 희석 반영

### 중기 개선

- 🔲 **산업별 특화 분석 템플릿** — 바이오, 게임, 금융 등 업종별 로직
- 🔲 **멀티 기업 비교 분석** — "삼성전자 vs SK하이닉스" 동시 비교
- 🔲 **해외 기업 분석 지원** — SEC(미국 공시), Bloomberg 데이터 연동

## 13. 이 프로젝트를 통해 배운 점

### AI는 계산기가 아니라 추론 도구

> 처음에는 AI에게 "IRR을 계산해줘"라고 시켰습니다. 틀렸습니다.

IRR(내부수익률)은 수학적으로 정확히 계산해야 하는 숫자입니다. AI에게 맡기면 "약 15% 정도 될 것 같습니다"처럼 부정확한 추정을 합니다. 이 프로젝트에서 **IRR 계산은 Python 코드로 정확히 수행하고, AI는 그 숫자의 의미를 해석하는 역할만** 담당하게 구조를 바꿨습니다.

→ **배움:** AI는 "추론과 서술"에 쓰고, 계산은 코드로 하라.

### 데이터 품질이 AI 품질을 결정한다

> 초기 시스템에서 삼성전자를 분석했더니 가상의 스타트업(Acme AI) 데이터가 결과에 섞였습니다.

원인: 검색 데이터베이스에 다른 회사 데이터가 있었고, 필터링이 없었습니다.
해결: `company_name` 메타데이터 필터링으로 다른 회사 데이터를 완전히 차단.

→ **배움:** AI 출력의 품질은 입력 데이터의 품질에 비례한다. "Garbage in, Garbage out."

### 기업 유형에 맞지 않는 가정은 비현실적 결과를 낸다

> 삼성전자에 스타트업 IRR 72%, MOIC 15x가 나왔습니다. 명백히 틀렸습니다.

원인: 시스템이 모든 기업을 같은 스타트업 파라미터로 계산했습니다.
해결: 기업 분류 레이어 추가 → 삼성전자: IRR 16%, MOIC 2.1x (현실적)

→ **배움:** AI 시스템도 도메인 지식(업계 상식)을 반영해야 현실적 결과가 나온다.

### RAG 없이는 AI가 근거 없는 주장을 한다

> RAG 없이 AI에게 "삼성전자 2023 영업이익은?"을 물으면 틀린 수치를 자신 있게 말합니다.

RAG(검색 후 생성)를 적용하면:
1. DART에서 실제 데이터 검색
2. "6,566,976,000,000원 (출처: DART 2023 사업보고서)"라고 정확한 근거와 함께 대답

→ **배움:** AI 분석 시스템에서 RAG는 선택이 아니라 필수다.

## 15. 실행 방법

### 사전 준비

- Python 3.11 이상 설치 ([python.org](https://www.python.org))
- DART API 키 발급 ([opendart.fss.or.kr](https://opendart.fss.or.kr/intro/main.do) — 무료)
- Anthropic API 키 발급 ([console.anthropic.com](https://console.anthropic.com) — 유료)

### 설치 및 실행

**① 저장소 다운로드**
```bash
git clone https://github.com/GitHaewon/ai-c-simulation.git
cd ai-c-simulation
```

**② 가상환경 생성 (프로젝트만을 위한 독립된 Python 환경)**
```bash
python -m venv .venv

# Windows
.venv\Scripts\activate

# Mac / Linux
source .venv/bin/activate
```

> 💡 가상환경이란? 프로젝트마다 다른 패키지 버전을 사용할 수 있게 하는 독립된 공간입니다.

**③ 필요한 패키지 설치**
```bash
pip install -r requirements.txt
```

> 💡 `requirements.txt`는 이 프로젝트에 필요한 모든 도구 목록입니다. 설치에 2~5분 걸릴 수 있습니다.

**④ API 키 설정**
```bash
# .env.example 파일을 복사해서 .env 파일 생성
copy .env.example .env     # Windows
cp .env.example .env       # Mac / Linux
```

`.env` 파일을 메모장으로 열고 실제 키 값 입력:
```
ANTHROPIC_API_KEY=sk-ant-...여기에_실제_키_입력
DART_API_KEY=...여기에_DART_키_입력
```

**⑤ 웹앱 실행**
```bash
streamlit run app.py
```

**⑥ 브라우저 접속**

터미널에 아래와 같이 표시되면 성공:
```
  You can now view your Streamlit app in your browser.
  Local URL: http://localhost:8501
```

브라우저에서 `http://localhost:8501` 접속

### 첫 실행 시 주의사항

- 첫 실행 시 AI 모델 파일 다운로드로 1~2분 추가 소요될 수 있습니다.
- DART API 키 없이도 실행은 되지만, 실제 공시 데이터 수집이 비활성화됩니다.
- Anthropic API 키 없이는 AI 토론 기능이 작동하지 않습니다.

### 빠른 테스트

설치 후 DART 연결 상태를 먼저 확인하세요:
```bash
python scripts/test_dart_and_entity_match.py
```
